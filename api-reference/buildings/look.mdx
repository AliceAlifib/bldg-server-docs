# Look at Floor

Get all buildings on a specific floor (direct children only).

## Endpoint

```
GET /v1/bldgs/look/:flr
```

The `:flr` parameter is the URI-encoded floor address.

## Example Request

```bash
# Buildings on the ground
curl https://bldg-server.fly.dev/v1/bldgs/look/g

# Buildings inside a specific building's floor 0
curl https://bldg-server.fly.dev/v1/bldgs/look/g%2Fb(10%2C20)%2Fl0
```

## Response

Returns an array of building objects (no wrapper). The first element is the container building itself, followed by its children:

```json
[
  {
    "address": "g",
    "name": "Ground",
    "entity_type": "ground",
    "is_composite": true,
    "visual_language": "{\"problem\":{\"3d_object\":\"livingRoomTable\",\"flr_height\":\"1.08\",\"flr0_height\":\"1.11\"},\"milestone\":{\"3d_object\":\"gardenRoundTable\",\"flr_height\":\"0.63\",\"flr0_height\":\"0.0\"},\"task\":{\"3d_object\":\"pottedPlant\"}}"
  },
  {
    "address": "g/b(10,20)",
    "bldg_url": "g/my-project",
    "name": "My Project",
    "entity_type": "problem",
    "is_composite": true,
    "x": 10,
    "y": 20,
    "flr": "g",
    "owners": ["alice@example.com"]
  }
]
```

## Notes

- The container building (the building that owns the floor) is included as the first element
- The container's `visual_language` field (JSON-encoded string) tells the client how to render each child's `entity_type` — see [Visual Language](/concepts/visual-language)
- Only returns direct children — use [scan](/api-reference/buildings/scan) for recursive lookup
