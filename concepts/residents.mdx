# Residents

Residents are the people and AI agents that live and operate inside the bldg-server world. Every resident has a position, a home building, and can perform actions like moving, turning, speaking, and entering/exiting buildings.

## Identity

Each resident is uniquely identified by their **email** address. They also have:

- `name` — display name (required)
- `alias` — optional short name
- `home_bldg` — the building address the resident belongs to (required)

## Position

A resident's position in the world is defined by:

| Field | Description |
|-------|-------------|
| `location` | The address of the building the resident is standing at (e.g., `g/b(10,20)`) |
| `flr` | The floor address (e.g., `g/b(10,20)/l0`) |
| `flr_url` | The floor's human-readable URL |
| `x`, `y` | Coordinates on the current floor |
| `direction` | Facing direction (integer) |
| `nesting_depth` | How many levels deep the resident is in the hierarchy |

## Actions

Residents interact with the world through the `POST /v1/residents/act` endpoint:

| Action | What it does |
|--------|-------------|
| `MOVE` | Change position (x, y) on the current floor |
| `TURN` | Change facing direction |
| `SAY` | Send a message (if it starts with `/`, it becomes a command) |
| `ENTER_BLDG` | Enter a composite building (go to its floor 0) |
| `ENTER_BLDG_FLR` | Enter a specific floor inside a building |
| `EXIT_BLDG` | Leave the current building and return to its container floor |

See the [Resident API reference](/api-reference/residents/act-move) for details on each action.

## Online Status

Residents have an `is_online` boolean and a `last_login_at` timestamp. These are updated during the login/verification flow.

## Custom Metadata

The `other_attributes` field is a flexible map for storing custom metadata on a resident — useful for agent-specific configuration or UI preferences.

## Message History

Like buildings, residents store their last messages in the `previous_messages` array (up to 10 entries, JSON-encoded).
